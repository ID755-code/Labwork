Data Security, Integrity, and Anomalies

Layers of Data Security

Securing data in a Database Management System (DBMS) involves multiple layers to protect against unauthorized access and breaches:

  • Authentication: Verifies the identity of users through methods like passwords, biometrics, or multi-factor authentication.
  • Authorization: Determines user permissions and roles, controlling access to different database operations.
  • Encryption: Secures data both at rest and in transit by making it unreadable without the decryption key.
  • Auditing and Monitoring: Tracks and logs database activities to detect and respond to security violations or breaches.

Layers of Data Integrity

Data integrity ensures accuracy, consistency, and reliability of data within a DBMS through various constraints and validation rules:

  • Entity Integrity: Requires each table to have a primary key to ensure unique and identifiable records.
  • Referential Integrity: Maintains consistent relationships between tables using valid foreign key references.
  • Domain Integrity: Ensures data entries conform to predefined rules or constraints, such as data types or ranges.
  • User-defined Integrity: Implements additional rules specific to business needs for further control over data validity.

Types of Anomalies in DBMS

Anomalies refer to inconsistencies or issues arising from improper database design or manipulation:

  • Update Anomalies: Occur when changes in one part of the database require corresponding updates in other parts, leading to discrepancies. For example, updating a customer's address in one record but not in all related records.
  • Insertion Anomalies: Arise when the database schema design makes it difficult to add new data without introducing inconsistencies or null values. For instance, requiring data to be inserted into multiple tables simultaneously.
  • Deletion Anomalies: Happen when removing data inadvertently causes the loss of other valuable information. For example, deleting a student record might also remove associated course information.

Normalization techniques can help address these anomalies by designing the database schema to minimize redundancy and ensure data consistency.
